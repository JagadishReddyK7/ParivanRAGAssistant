Parivahan RAG Assistant

Technical Report ML Engineering  ·  May 2026  ·  Sentence Transformers · FAISS · Gemini 1.5 Flash · Streamlit

1. Introduction

India's motor vehicle services portal, Parivahan (parivahan.gov.in), publishes official procedural PDFs for Driving Licence (DL) and Registration Certificate (RC) applications through its Sarathi and Vahan sub-systems. Citizens frequently struggle to navigate these documents, leading to avoidable RTO visits and errors in applications.

This project builds a Retrieval-Augmented Generation (RAG) chatbot that ingests official Parivahan PDFs, indexes them in a local FAISS vector store, and uses a free LLM (Gemini 1.5 Flash or Groq Llama 3) to answer citizen queries with grounded, cited answers — with optional Hindi response mode.

4. Data

PDFs are fetched from official Parivahan sub-portals at runtime. The parser (PyMuPDF / fitz) extracts plain text page-by-page, normalises whitespace, and splits into 500-word chunks with a 50-word sliding overlap to preserve cross-boundary context.

A fallback knowledge base (6 manually curated chunks) activates when PDFs are unavailable (network restrictions, URL changes), ensuring the demo always works.

SourceContentChunks (est.)
DL_Procedure.pdfLearner licence, permanent DL, fees, forms~20
RC_Procedure.pdfNew registration, transfer, NOC, fees~18
Fallback KBCurated DL+RC summary chunks6

5. Method

5.1 Embeddings

sentence-transformers/all-MiniLM-L6-v2 produces 384-dimensional sentence embeddings, CPU-only, ~80 MB, with excellent semantic retrieval quality for English and transliterated Hindi. Embeddings are L2-normalised, enabling cosine similarity via FAISS IndexFlatIP (inner product).

5.2 Vector Store

FAISS IndexFlatIP performs exact nearest-neighbour search. For the expected corpus size (~50–200 chunks), exhaustive search is both fast (<5ms) and optimal — approximate methods (IVF, HNSW) are unnecessary. The index is persisted to disk and loaded at startup.

5.3 Prompt Engineering

The system prompt instructs the LLM to: (a) answer only from provided context, (b) cite sources using [SOURCE - page X] format, (c) avoid hallucinating form numbers or fees, and (d) optionally respond in Hindi. Top-K retrieved chunks are injected as labelled context blocks.

5.4 LLM

Primary: Gemini 1.5 Flash (free tier, 15 RPM). Fallback: Groq Llama3-8b-8192 (free tier, 30 RPM). Both accept a system message + user message pattern. The provider is switchable via LLM_PROVIDER env var.

Latency

  • Index build (50 chunks, CPU): ~4 seconds
  • Query embedding: ~30ms
  • FAISS search (50 vectors): <1ms
  • LLM response (Gemini Flash): 1–3 seconds

7. Limitations

  • PDF availability: Parivahan URLs change; fallback KB is static and may become outdated.
  • Hindi input: Query language is English-only; Hindi toggle only affects output.
  • Hallucination risk: LLM may interpolate facts not in context despite the system prompt guard.
  • Corpus coverage: Only two source PDFs — many Parivahan procedures are not yet indexed.
  • Free API limits: Gemini Flash: 15 RPM; Groq Llama3: 30 RPM. Not suitable for high traffic.

8. Ablation Study

Finding: top-K=5 with 500-word overlapping chunks provides the best quality/latency trade-off.

9. Deployment Steps

Local (1 command)

git clone <repo> && cd parivahan_rag
cp .env.example .env        # add GEMINI_API_KEY
bash run.sh                 # installs deps, builds index, launches UI

Hugging Face Spaces

1. Create new Space → SDK: Streamlit
2. Push code (README.md already has HF metadata)
3. Settings → Secrets: add GEMINI_API_KEY
4. Space auto-builds; index generated on first query

Streamlit Cloud

1. Connect GitHub repo at share.streamlit.io
2. Main file: app.py
3. Secrets: paste .env contents
4. Deploy → first run builds index (~30s)

Docker

docker build -t parivahan-rag .
docker run -p 8501:8501 -e GEMINI_API_KEY=xxx parivahan-rag
